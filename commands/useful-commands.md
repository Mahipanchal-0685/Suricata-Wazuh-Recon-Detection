# Useful Commands

This document contains key commands used during the deployment, validation, and troubleshooting of the Suricata-Wazuh reconnaissance detection project.

---

## Nmap Reconnaissance Scan

Used from the Kali Linux attacker machine to generate reconnaissance traffic.

```bash
sudo nmap -A <target-ip>
```

---

## Verify Suricata Service Status

Check whether the Suricata IDS service is running correctly.

```bash
sudo systemctl status suricata
```

---

## Validate Suricata Configuration

Validate the Suricata configuration file before restarting the service.

```bash
sudo suricata -T -c /etc/suricata/suricata.yaml
```

---

## Identify Active Network Interface

Used to determine which network interface Suricata should monitor.

```bash
ip a
```

---

## Monitor Suricata Events

View real-time events generated by Suricata.

```bash
sudo tail -f /var/log/suricata/eve.json
```

---

## Verify Wazuh Manager Status

Confirm that the Wazuh Manager service is operational.

```bash
sudo systemctl status wazuh-manager
```

---

## Restart Wazuh Manager

Restart the manager after configuration changes.

```bash
sudo systemctl restart wazuh-manager
```

---

## Verify Suricata Alerts in Wazuh

Check whether Suricata alerts have been ingested by Wazuh.

```bash
sudo grep -i suricata /var/ossec/logs/alerts/alerts.json | tail -10
```

---

## Check Suricata Log Monitoring

Confirm that Wazuh is monitoring the Suricata eve.json log file.

```bash
sudo grep -a "eve.json" /var/ossec/logs/ossec.log
```

---
